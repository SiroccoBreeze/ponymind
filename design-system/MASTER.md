## 1. 风格选择与理由

### 选定风格：**Glassmorphism-lite × Soft UI Evolution**（命名：*AI Soft Glass*）

| 候选方案 | 评分 | 淘汰/选择原因 |
|---|---|---|
| Pure Neumorphism | ✗ | WCAG AA 对比度不足，暗色模式几乎无法实现 |
| Cyberpunk / Cyber-Kawaii | ✗ | 霓虹过曝，干扰阅读，与知识平台定位不符 |
| Glassmorphism-lite + Soft UI Evolution | ✅ **选定** | WCAG AA+ 可达，深浅模式天然支持，提供"轻玻璃深度感"而不刺眼 |

**AI Soft Glass 核心定义：**
- **基底**：Soft UI Evolution — 改进对比度的软阴影，8–16px 圆角，200–300ms 过渡
- **点缀**：Glassmorphism — 用于 Navbar、Modal、悬浮卡片（backdrop-blur + 半透明边框）
- **发光**：极克制的 micro-glow（仅用于主按钮、徽章 hover、AI 回答区域边框）
- **禁止**：纯黑背景 + 全霓虹、连续闪光动画、scale transform 引起的布局抖动

## 3. 字体系统

### 3.1 字体选择

| 角色 | 字体 | 理由 |
|---|---|---|
| **Heading** | `Space Grotesk` | 几何感现代标题字，适合 AI/科技产品，有独特字符但不怪异 |
| **Body** | `DM Sans` | 极高可读性，友好不刻板，适合长文阅读（Q&A 回答区域关键） |
| **Code** | `Fira Code` | 连字支持，开发者常见选择，暗色代码块极佳 |

```css

const spaceGrotesk = Space_Grotesk({
  subsets: ['latin'],
  variable: '--font-heading',
  weight: ['400', '500', '600', '700'],
  display: 'swap',
});

const dmSans = DM_Sans({
  subsets: ['latin'],
  variable: '--font-body',
  weight: ['400', '500', '700'],
  display: 'swap',
});

// html className: `${spaceGrotesk.variable} ${dmSans.variable}`
```

- 所有 `h1–h3`：`font-heading` + 对应字重
- 正文、UI 文字：`font-sans`（DM Sans）

## 7. 动效规范

### 7.1 时间与缓动

| 用途 | 时长 | 缓动 |
|---|---|---|
| micro（图标 hover、色彩变化）| 150ms | `ease-out` |
| default（卡片 hover、按钮状态）| 200ms | `ease-out` |
| complex（侧边栏展开、模态框）| 300ms | `cubic-bezier(0.4, 0, 0.2, 1)` |
| page transition | 250ms | `ease-in-out` |

### 7.2 常用 Tailwind 类

```
transition-colors duration-150 ease-out       /* 颜色过渡 */
transition-all duration-200 ease-out          /* 综合过渡（卡片）*/
transition-transform duration-200 ease-out   /* 位移/缩放 */
hover:-translate-y-0.5                        /* 卡片轻微上浮（不超过 2px）*/
hover:shadow-md                               /* 阴影加深 */
```

### 7.3 Reduced Motion 规则（必须）

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

### 7.4 禁止的动效

- `animate-bounce` 用于装饰性元素
- `scale > 1.05` 用于卡片 hover（会引起布局漂移）
- 超过 500ms 的非 loading 动画
- 连续循环的背景动画（粒子、波浪等）

## 10. 深色模式规范

- 使用 `next-themes` + Tailwind `class` 策略（已存在）
- 深色背景非纯黑：`#0D0D1A`（深蓝黑，更易读）
- 所有 `rgba` 透明色在深色模式下需重新测算对比度
- 深色模式卡片边框：`rgba(255,255,255,0.08)` — 确保可见
- 深色代码块：独立配色，不依赖 `invert`
- `color-scheme: dark` 需在 html 标签设置（影响滚动条、表单元素原生色）

## 11. 可访问性规范（WCAG AA 强制）

| 规则 | 要求 |
|---|---|
| 正文文字对比度 | ≥ 4.5:1 |
| 大字 (18px+/bold 14px+) 对比度 | ≥ 3:1 |
| focus 指示器 | `focus-visible:ring-2 ring-primary ring-offset-2`，不可只用 `outline-none` |
| 键盘导航 | Tab 顺序匹配视觉顺序，无键盘陷阱 |
| Skip Link | 页面顶部提供"跳转到主内容"隐藏链接 |
| 图片 alt | 所有 `<Image>` 必须有描述性 alt |
| 表单 label | 所有 input 必须有关联 label 或 aria-label |
| 颜色非唯一 | 不能只靠颜色区分错误/成功（需加图标或文字）|
| Reduced Motion | 必须实现 `prefers-reduced-motion` 媒体查询 |
| 触摸目标 | 最小 44×44px（移动端所有可点击元素）|

## 13. 图标规范

- **统一图标库**：`lucide-react`（已安装，轻量 SVG）
- **禁止**：用 emoji 作为功能性图标
- **尺寸规范**：
  - 行内图标（文字旁）：`w-4 h-4`（16px）
  - 按钮图标：`w-4 h-4`
  - 卡片图标：`w-5 h-5`（20px）
  - 大型展示图标：`w-6 h-6`（24px）
- **统一 stroke**：默认 `strokeWidth={1.5}`（不用 Lucide 的 2，更细腻）
- **AI 专属图标**：`Sparkles`（AI 标注）、`BrainCircuit`（AI 回答）、`Wand2`（AI 生成）

## 14. Pre-Delivery Checklist

在提交任何页面/组件前，验证以下项目：

### Visual
- [ ] 未使用 emoji 作为功能图标，全部替换为 Lucide SVG
- [ ] 图标尺寸一致（对应规范尺寸）
- [ ] 浅色/深色模式均视觉测试通过
- [ ] 卡片在浅色模式下边框/背景可见（不用 rgba 太低的透明色）

### Interaction
- [ ] 所有可点击元素有 `cursor-pointer`
- [ ] Hover 提供视觉反馈（颜色/阴影/轻微位移）
- [ ] Focus 状态可见（`focus-visible:ring-2`）
- [ ] 过渡时间 150–300ms，禁止 linear 缓动

### Accessibility
- [ ] 文字对比度 ≥ 4.5:1（使用 WebAIM Contrast Checker 验证）
- [ ] 所有图片有 alt
- [ ] 所有表单元素有 label
- [ ] `prefers-reduced-motion` 已处理
- [ ] 键盘可导航（Tab 可到达所有交互元素）

### Layout
- [ ] 响应式验证：375px、768px、1024px、1440px
- [ ] 无横向滚动（移动端）
- [ ] 固定 Navbar 高度 64px，页面 `pt-16` 或 `mt-16` 避免内容被遮挡
- [ ] 最大宽度 `max-w-7xl` 居中

### Performance
- [ ] 动态内容使用 Skeleton 占位，避免 CLS
- [ ] 图片使用 `next/image` with `fill` 或明确 `width/height`
- [ ] 字体使用 `next/font` 异步加载，`display: swap`
